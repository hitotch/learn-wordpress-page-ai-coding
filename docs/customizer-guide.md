# カスタマイザー設定ガイド

GeneratePress（子テーマ）のカスタマイザーを使って、サイトの構造を設定する方法を解説します。

## カスタマイザーとは

WordPressのカスタマイザーは、管理画面からサイトの見た目や構造をリアルタイムプレビューしながら設定できる機能です。

> **💡 AIとカスタマイザーの関係**: AIコーディングツールはWP-CLIを使ってカスタマイザーの一部設定を変更することもできますが、リアルタイムプレビューで確認しながら調整できる管理画面GUIのほうが直感的です。カスタマイザーの設定は管理画面で行うことを推奨します。

**アクセス方法**: 管理画面 → 外観 → カスタマイズ（または `http://localhost/wp-admin/customize.php`）

## このプロジェクトでのカスタマイザーの使い方

### 重要な考え方

カスタマイザーでは**構造設定のみ**を行います。色やフォントなどのビジュアルデザインは子テーマの `style.css` で管理します。

| カスタマイザーで設定するもの | style.cssで管理するもの |
|---|---|
| ヘッダーのレイアウト型 | 色、フォントサイズ |
| サイドバーの有無・位置 | 余白、背景色 |
| コンテナの幅 | ボタンのデザイン |
| フッターのウィジェットエリア数 | アニメーション |

### GeneratePress で設定できる主な項目

| カテゴリ | 設定項目 | 説明 |
|---|---|---|
| **Layout** | Container Width | サイト全体のコンテンツ幅 |
| **Layout** | Content Layout | デフォルトのコンテンツレイアウト |
| **Header** | Header Preset | ヘッダーのレイアウトパターン |
| **Primary Navigation** | Navigation Location | ナビゲーションの表示位置 |
| **Blog** | Content Type | ブログ一覧の表示形式 |
| **Footer** | Footer Widgets | フッターウィジェットの列数 |
| **Footer** | Back to Top | トップに戻るボタンの表示 |

## 生成AIを活用したカスタマイザー設定

### 設定を相談するプロンプト例

以下のようなプロンプトをAIに送ると、適切なカスタマイザー設定をアドバイスしてもらえます。

#### サイト構造の相談

```
GeneratePress（無料版）のカスタマイザーで、以下のようなサイトを構築したいです。
適切な設定値を教えてください。

サイトの種類: [コーポレートサイト / ポートフォリオ / ブログ など]
ヘッダー: [ロゴ左・ナビ右 / センタリング など]
サイドバー: [なし / 右サイドバー など]
コンテンツ幅: [狭め / 標準 / 広め]
フッター: [シンプル / ウィジェット3列 など]
```

#### 特定の設定について質問

```
GeneratePress のカスタマイザーで「Container Width」を設定したいのですが、
コーポレートサイトに適した幅はどのくらいですか？
現在のデフォルト値は 1170px です。
```

## 設定のエクスポートとインポート

カスタマイザーの設定は、プラグインを使ってエクスポート/インポートできます。これにより、ローカルの設定を本番サーバーに簡単に適用できます。

### 準備: Customizer Export/Import プラグイン

1. 管理画面 → プラグイン → 新規追加
2. 「Customizer Export/Import」で検索
3. インストール → 有効化

### エクスポート手順

1. 管理画面 → 外観 → カスタマイズ
2. 左パネルの最下部にある「Export/Import」をクリック
3. 「Export」ボタンをクリック
4. `.dat` ファイルがダウンロードされる

### インポート手順（本番サーバー）

1. 本番のWordPress管理画面 → 外観 → カスタマイズ
2. 「Export/Import」→「Import」
3. エクスポートした `.dat` ファイルを選択
4. 「Import」ボタンをクリック

> 本番サーバーにも同じく「Customizer Export/Import」プラグインが必要です。

## 注意点

- カスタマイザーの設定はデータベースに保存されるため、`docker compose down` でデータベースを削除すると設定も消えます
- 重要な設定をした後は、エクスポートしてバックアップしておくと安心です
- GeneratePress Pro の機能はカスタマイザーに表示されますが、このプロジェクトでは使用しません

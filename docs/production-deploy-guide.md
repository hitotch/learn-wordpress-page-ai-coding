# 本番サーバーへの適用ガイド

ローカル開発環境で作成したデザインやページを、本番のWordPressに適用する手順を解説します。

## 前提

- **本番のWordPressはレンタルサーバーのインストール機能等で構築済み**であること
- この手順は、レンタルサーバー（Xserver, さくら, ConoHa等）でもVPSでも、WordPressが動いている環境なら共通で使えます

## 適用の全体像

ローカルで作成するものは3種類あり、それぞれ本番への適用方法が異なります。

```
┌─────────────────────────────────────────────────┐
│ ローカル開発環境                                    │
│                                                   │
│  ① 子テーマ（CSS / functions.php）                  │
│     → ファイルをアップロード                         │
│                                                   │
│  ② カスタマイザー設定                               │
│     → エクスポート → インポート                      │
│                                                   │
│  ③ ページ（ブロックマークアップ）                    │
│     → エクスポート → インポート                      │
│                                                   │
└───────────────────┬─────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────┐
│ 本番WordPress（レンタルサーバー等）                  │
│                                                   │
│  ① GeneratePress + 子テーマ 有効                    │
│  ② カスタマイザー設定 適用済み                       │
│  ③ ページ インポート済み                            │
│                                                   │
│  → ローカルと同じデザインが再現される               │
└─────────────────────────────────────────────────┘
```

## Step 1: 本番にGeneratePress / GenerateBlocksをインストール

まず、本番のWordPressに必要なテーマとプラグインをインストールします。

### GeneratePress テーマ

1. 本番の管理画面 → 外観 → テーマ → 新しいテーマを追加
2. 「generatepress」で検索
3. 「インストール」→ まだ有効化しない（子テーマを先に準備する）

### GenerateBlocks プラグイン

1. 本番の管理画面 → プラグイン → 新規追加
2. 「generateblocks」で検索
3. 「今すぐインストール」→「有効化」

## Step 2: 子テーマを本番に適用

ローカルで開発した子テーマ（`generatepress-child/` フォルダ）を本番に適用します。

### 方法A: ZIPファイルでアップロード（おすすめ）

1. **子テーマフォルダをZIP化**

   ```bash
   # ワークスペースルートで実行
   cd generatepress-child
   zip -r ../generatepress-child.zip .
   ```

   または、ファイルマネージャーやFinderで `generatepress-child` フォルダをZIP圧縮。

2. **本番の管理画面からアップロード**
   - 外観 → テーマ → 新しいテーマを追加 → テーマのアップロード
   - ZIPファイルを選択 → 「今すぐインストール」
   - 「有効化」をクリック

### 方法B: FTP / ファイルマネージャーでアップロード

1. FTPクライアント（FileZilla等）またはレンタルサーバーのファイルマネージャーを開く
2. `wp-content/themes/` ディレクトリに移動
3. `generatepress-child/` フォルダをそのままアップロード
4. 管理画面 → 外観 → テーマ → 子テーマを「有効化」

### 子テーマを更新する場合

CSSやfunctions.phpを更新した場合：

- **方法A**: ZIPファイルを再作成して、管理画面から上書きインストール
- **方法B**: FTPで変更したファイルだけを上書きアップロード（おすすめ）

## Step 3: カスタマイザー設定を移行

### 準備

ローカル・本番の両方に「Customizer Export/Import」プラグインをインストールします。

1. プラグイン → 新規追加 → 「Customizer Export/Import」で検索
2. インストール → 有効化

### エクスポート（ローカル）

1. ローカルの管理画面 → 外観 → カスタマイズ
2. 左パネル最下部の「Export/Import」
3. 「Export」ボタンをクリック
4. `.dat` ファイルがダウンロードされる

### インポート（本番）

1. 本番の管理画面 → 外観 → カスタマイズ
2. 「Export/Import」→「Import」
3. `.dat` ファイルを選択 → 「Import」
4. カスタマイザー設定が本番に反映される

## Step 4: ページを移行

### 方法A: WordPress標準のエクスポート/インポート

#### エクスポート（ローカル）

1. ローカルの管理画面 → ツール → エクスポート
2. 「固定ページ」を選択
3. 「エクスポートファイルをダウンロード」
4. `.xml` ファイルがダウンロードされる

#### インポート（本番）

1. 本番の管理画面 → ツール → インポート
2. 「WordPress」の「今すぐインストール」（初回のみ）
3. 「インポーターの実行」
4. `.xml` ファイルを選択してアップロード

### 方法B: コードエディターでコピー＆ペースト

少数のページであれば、ブロックマークアップを直接コピーする方が簡単な場合もあります：

1. ローカルの管理画面 → 固定ページ → 対象ページの編集
2. 「⋮」→「コードエディター」→ マークアップをすべてコピー
3. 本番の管理画面 → 固定ページ → 新規追加
4. 「コードエディター」→ マークアップを貼り付け → 「公開」

### 方法C: WP-CLIでマークアップを取得

AIコーディングツールでページを作成した場合、WP-CLIでマークアップを直接取得できます：

```bash
# ページ一覧を確認
docker compose exec wpai-wordpress wp post list --post_type=page --path=/var/www/html --allow-root

# 特定ページのマークアップを取得
docker compose exec wpai-wordpress wp post get <ページID> --field=post_content --path=/var/www/html --allow-root
```

取得したマークアップを本番の管理画面のコードエディターに貼り付けてください。

## 注意点とトラブルシューティング

### 画像のURL

ローカルのマークアップ内の画像URLは `http://localhost/wp-content/uploads/...` になっています。本番に移行する際は、画像を本番にアップロードしてURLを差し替える必要があります。

対処法：
1. 本番の管理画面 → メディア → 新規追加 で画像をアップロード
2. アップロードした画像のURLをコピー
3. マークアップ内のURLを差し替え

### GenerateBlocks のバージョン

ローカルと本番で GenerateBlocks のバージョンが大きく異なると、ブロックの互換性に問題が出る場合があります。両方で同じバージョンを使用することを推奨します。

### CSSが反映されない場合

- ブラウザのキャッシュをクリアしてください
- 子テーマが正しく有効化されているか確認してください
- FTPでアップロードした場合、ファイルのパーミッションを確認してください（通常は 644）

## まとめ: 適用チェックリスト

本番に適用する際のチェックリストです：

- [ ] GeneratePress テーマをインストール
- [ ] GenerateBlocks プラグインをインストール＆有効化
- [ ] 子テーマをアップロード＆有効化
- [ ] Customizer Export/Import プラグインをインストール
- [ ] カスタマイザー設定をインポート
- [ ] ページをインポート（またはコードエディターでコピー）
- [ ] 画像をアップロードしてURLを差し替え
- [ ] ブラウザで表示を確認
